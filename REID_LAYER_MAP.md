# REID Layer Map
### A visual and structural breakdown of the full REID Cognitive Architecture

The REID Cognitive Architecture is built as a layered system.  
Each layer builds on the one below it, creating a stable, universal,
drift‑resistant reasoning framework for humans and machines.

This document provides a high‑level map of all layers in REID.

---

# 1. Layer 1 — The Substrate (Foundation Layer)

**Purpose:** Provide a universal structural base for all reasoning.

**Components:**
- State  
- Forces  
- Horizon  
- Flows  

**Properties:**
- Domain‑agnostic  
- Stable across contexts  
- Universal  
- Drift‑resistant  

Everything in REID sits on the substrate.

---

# 2. Layer 2 — Invariant Interaction Layer

**Purpose:** Define how the invariants interact.

**Core Loop:**
**Horizon → Forces → Flows → State → Horizon**

**Functions:**
- Explains movement  
- Predicts change  
- Reveals bottlenecks  
- Anchors planning  

This is the engine of REID.

---

# 3. Layer 3 — Reasoning Modes Layer

**Purpose:** Provide two complementary reasoning modes.

### **Mode N (Normal)**
- Expanded  
- Exploratory  
- Full‑context  

### **Mode T (Tight)**
- Compressed  
- Compute‑efficient  
- Drift‑resistant  
- Minimal steps  

Mode T is the shared compression mode for humans and machines.

---

# 4. Layer 4 — Domain Translation Layer

**Purpose:** Move reasoning between domains without drift.

**Process:**
1. Extract invariants  
2. Remove domain language  
3. Re‑express structure in the target domain  

**Outcome:**  
Structure stays constant.  
Only language changes.

---

# 5. Layer 5 — Multi‑Human Node Layer

**Purpose:** Model multiple humans structurally.

Each human is a node with:
- State  
- Forces  
- Horizon  
- Flows  

**Enables:**
- Alignment without agreement  
- Conflict mapping  
- Horizon comparison  
- Flow balancing  

This is the human‑systems layer of REID.

---

# 6. Layer 6 — Compute Model Layer

**Purpose:** Treat compute as a flow resource.

**Includes:**
- Human compute  
- Machine compute  
- Flow constraints  
- Cost‑aware reasoning  
- Mode T optimization  

This layer unifies human and machine cognition.

---

# 7. Layer 7 — Long‑Arc Reasoning Layer

**Purpose:** Support multi‑step, multi‑domain, multi‑human planning.

**Focus:**
- Structural trajectories  
- Horizon stability  
- Force alignment  
- Flow allocation  
- Drift prevention  

This is the strategic layer of REID.
