# REID Advanced FAQ
### Deep technical questions about the REID Cognitive Architecture

This FAQ addresses advanced, structural, and architectural questions that
arise when implementing, extending, or researching REID.

---

## 1. Is REID a reasoning framework or a cognitive architecture?
REID is a **cognitive architecture**.

A reasoning framework provides techniques.  
A cognitive architecture provides:
- Structure  
- Layers  
- Invariants  
- Interaction rules  
- Translation mechanisms  
- Multi‑human modeling  
- Compute alignment  

REID is a full architecture, not a toolkit.

## 2. Why does REID use invariants instead of domain rules?
Domain rules drift.  
Invariants do not.

Invariants appear in every system:
- State  
- Forces  
- Horizon  
- Flows  

This makes REID:
- Universal  
- Drift‑resistant  
- Domain‑agnostic  
- Scalable  

Invariants are the backbone of the substrate.
