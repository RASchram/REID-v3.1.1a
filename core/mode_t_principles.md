# Principles of Mode T
### The compressed reasoning mode of the REID Cognitive Architecture

Mode T is REID’s tight, compressed, compute‑efficient reasoning mode.
It reduces drift, shortens reasoning chains, and increases clarity by
removing unnecessary expansion.

This document explains the principles behind Mode T and how it differs
from Mode N.

---

## 1. Compression Over Expansion
Mode N expands reasoning to explore the full structure.

Mode T compresses reasoning to:
- Reduce cognitive load  
- Minimize compute usage  
- Increase clarity  
- Avoid drift  

Compression is not loss — it is **precision**.

---

## 2. Structural Priority
Mode T prioritizes the invariants:
- State  
- Forces  
- Horizon  
- Flows  

Everything else is secondary.

If a detail does not change an invariant, Mode T discards it.

---

## 3. Minimal Steps
Mode T aims for the **fewest steps** needed to reach a stable,
structural conclusion.

This reduces:
- Loops  
- Redundant reasoning  
- Over‑explanation  
- Noise  

Mode T is the shortest path through the invariants.

---

## 4. Compute as a Flow
Mode T treats compute as a **flow resource**.

This means:
- Every step has a cost  
- Every loop is a drain  
- Every expansion must justify itself  

Mode T optimizes reasoning the same way REID optimizes any flow.

---

## 5. Drift Prevention
Drift occurs when reasoning:
- Wanders  
- Over‑expands  
- Follows irrelevant branches  
- Loses structural alignment  

Mode T prevents drift by:
- Anchoring to invariants  
- Compressing steps  
- Removing domain noise  

---

## 6. Horizon Lock
Mode T keeps the horizon fixed and clear.

If the horizon is ambiguous, Mode T:
- Clarifies it  
- Reduces it  
- Or selects the minimal viable horizon  

A drifting horizon produces drifting reasoning.

---
