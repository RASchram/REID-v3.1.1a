# Invariant Interaction Map
### How State, Forces, Horizon, and Flows interact in the REID Cognitive Architecture

The four invariants of REID — **State, Forces, Horizon, Flows** — do not
operate independently. They form a structural system where each
invariant shapes, constrains, and enables the others.

This document maps their interactions and shows how they create a stable
reasoning substrate.

---

# 1. Overview of the Interaction System

The invariants interact in a continuous loop:

**Horizon → Forces → Flows → State → (back to Horizon)**

Each invariant influences the next, creating a structural cycle that
governs all reasoning, movement, and change.

---

# 2. Horizon → Forces
The horizon determines:
- Which forces matter  
- Which forces strengthen  
- Which forces weaken  
- Which forces are irrelevant  

A clear horizon organizes forces.  
A vague horizon amplifies noise.

---

# 3. Forces → Flows
Forces shape how flows behave by:
- Accelerating flows  
- Blocking flows  
- Redirecting flows  
- Increasing flow friction  
- Creating flow bottlenecks  

Strong forces require strong flows.  
Weak flows amplify the impact of forces.

---

# 4. Flows → State
Flows determine:
- How quickly state can change  
- Whether change is possible  
- How stable the state becomes  
- Whether the system drifts or stabilizes  

Flow scarcity slows state transitions.  
Flow abundance accelerates them.

---

# 5. State → Horizon
State determines:
- Which horizons are realistic  
- Which horizons are impossible  
- Which horizons need adjustment  
- How far the horizon can be set  

A stable state supports long horizons.  
An unstable state requires short horizons.

---

# 6. Cross‑Invariant Interactions

## A. Forces ↔ State
Forces act on state.  
State determines how forces are absorbed or resisted.

## B. Horizon ↔ Flows
The horizon defines where flows should go.  
Flows determine whether the horizon is reachable.
