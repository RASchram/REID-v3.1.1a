# REID Examples Overview
### How to use the examples in the REID Cognitive Architecture

The `/examples` directory contains practical demonstrations of how to
apply REID’s structural reasoning across domains, contexts, and
multi‑human scenarios. This overview explains the purpose of each
example category and how to use them effectively.

---

## 1. Purpose of the Examples
The examples are designed to:
- Show REID in action  
- Demonstrate invariant‑based reasoning  
- Provide domain‑agnostic templates  
- Teach Mode T and Mode N through contrast  
- Illustrate multi‑human alignment  
- Show compute‑efficient reasoning patterns  

Each example is intentionally simple, structural, and easy to adapt.

---

## 2. Example Categories

### **A. Reasoning Efficiency Examples**
These examples demonstrate:
- Mode N vs Mode T  
- Compression  
- Drift reduction  
- Force ranking  
- Horizon locking  

They show how REID produces clearer, shorter, more stable reasoning.

---

### **B. Domain Translation Examples**
These examples show how REID moves reasoning between domains without drift.

Examples include translations such as:
- Engineering → Parenting  
- Leadership → Health  
- Strategy → Relationships  

Each example follows the same pattern:
1. Extract invariants  
2. Strip domain language  
3. Re‑express in the target domain  

This teaches structural equivalence.

---

### **C. Multi‑Human Node Examples**
These examples demonstrate:
- Modeling each human as a node  
- Mapping horizons  
- Identifying force conflicts  
- Tracking flows between people  
- Finding alignment without forcing agreement  

These are ideal for:
- Teams  
- Families  
- Partnerships  
- Leadership scenarios  

---

### **D. Compute‑Efficiency Examples**
These examples show how REID:
- Treats compute as a flow  
- Reduces chain‑of‑thought length  
- Avoids unnecessary loops  
- Minimizes cognitive load  
- Optimizes reasoning for cost  

Useful for both human and machine reasoning.

---

### **E. Real‑World Scenarios**
These examples apply REID to:
- Planning  
- Decision‑making  
- Conflict resolution  
- Strategy  
- Communication  
- Long‑arc reasoning  

They show how REID works in everyday life.

---

## 3. How to Use the Examples
Each example is structured so you can:
1. Read the scenario  
2. Identify the invariants  
3. Follow the reasoning path  
4. Compare Mode N vs Mode T  
5. Apply the pattern to your own domain  

The examples are templates, not scripts.

---

## 4. Creating Your Own Examples
To create your own REID example:
1. Choose a scenario  
2. Extract the invariants  
3. Identify forces and flows  
4. Fix the horizon  
5. Reason in Mode N  
6. Compress to Mode T  
7. (Optional) Translate to another domain  

This process builds structural fluency.

---

## 5. Why the Examples Matter
The examples are where REID becomes real.

They:
- Teach the architecture  
- Show the invariants in action  
- Demonstrate universality  
- Build intuition  
- Reveal the power of structural reasoning  

The examples are the bridge between theory and practice.
