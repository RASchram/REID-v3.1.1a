# REID Architecture Overview
### A high‑level map of the REID Cognitive Architecture

REID is a universal cognitive architecture built on structural
invariants that appear across all domains. It provides a stable,
drift‑resistant reasoning substrate for humans and machines, enabling
clarity, alignment, and long‑arc planning.

This document summarizes the full architecture at a high level.

---

# 1. The Structural Foundation: The Substrate

At the base of REID is the **substrate** — a domain‑agnostic structural
layer composed of four invariants:

- **State** — what exists now  
- **Forces** — what acts on it  
- **Horizon** — where it is going  
- **Flows** — what moves through it  

These invariants appear in every system, human or technical.  
They form the universal reasoning lens of REID.

---

# 2. The Invariant Interaction Loop

The invariants interact in a continuous structural cycle:

**Horizon → Forces → Flows → State → Horizon**

This loop governs:
- Movement  
- Change  
- Stability  
- Drift  
- Planning  
- Alignment  

It is the engine of REID.

---

# 3. Reasoning Modes: Mode N and Mode T

REID uses two reasoning modes:

### **Mode N — Normal Reasoning**
- Expanded  
- Exploratory  
- Full‑context  
- Useful for discovery  

### **Mode T — Tight Reasoning**
- Compressed  
- Compute‑efficient  
- Drift‑resistant  
- Minimal steps  
- Ideal for clarity and alignment  

Mode T is the shared compression mode for humans and machines.

---

# 4. Domain Translation Layer

REID separates **structure** from **content**.

This allows reasoning to move between domains without drift:
- Engineering → Parenting  
- Leadership → Health  
- Strategy → Relationships  

Translation steps:
1. Extract invariants  
2. Remove domain language  
3. Re‑express structure in the target domain  

Structure stays constant.  
Only language changes.

---

# 5. Multi‑Human Node Model

Each human is modeled as a structural node with their own:
- State  
- Forces  
- Horizon  
- Flows  

This enables:
- Alignment without agreement  
- Conflict mapping  
- Horizon comparison  
- Flow balancing  
- Long‑arc coordination  

It is the backbone of REID’s human‑systems reasoning.

---

# 6. Compute Model

REID treats compute as a **flow resource**.

This enables:
- Efficient reasoning  
- Cost‑aware planning  
- Cloud‑aligned structure  
- Mode T compression  
- Scalable cognition  

Human compute and machine compute are modeled the same way.

---

# 7. Long‑Arc Reasoning

REID is designed for:
- Multi‑step planning  
- Multi‑domain reasoning  
- Multi‑human coordination  
- Multi‑year trajectories  

It avoids short‑term noise and focuses on structural movement.

The architecture is designed for real‑world use.
