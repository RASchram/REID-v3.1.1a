# Contributing to REID
### Guidelines for contributing to the REID Cognitive Architecture

Thank you for your interest in contributing to REID.  
This project is structured, documented, and versioned to support
long‑term clarity and stability.  
The following guidelines help maintain consistency across contributions.

---

## 1. Philosophy
REID is a cognitive architecture built on:
- Structural invariants
- Cross‑domain reasoning
- Multi‑human node modeling
- Compute‑efficient reasoning
- Long‑arc planning

All contributions should preserve these principles.

---

## 2. Repository Structure
- `/docs` — Core architecture documents  
- `/examples` — Demonstrations and applied scenarios  
- `/vision` — Long‑arc integration and ecosystem alignment  
- `README.md` — Overview and onboarding  
- `ROADMAP.md` — Future development plan  
- `CHANGELOG.md` — Version history  

Please place new files in the correct folder.

---

## 3. Writing Standards
All documentation should:
- Use clear, structured Markdown
- Follow REID’s invariant terminology
- Avoid domain‑specific jargon unless necessary
- Maintain conceptual consistency across files

---

## 4. Versioning
All changes must be reflected in:
- `CHANGELOG.md`  
- `ROADMAP.md` (if relevant)

Version numbers follow:
`major.minor.patch`  
Example: `3.1.1a`

---

## 5. Pull Requests
When submitting a PR:
- Describe the purpose of the change
- Reference related issues or roadmap items
- Keep changes focused and minimal
- Ensure all Markdown renders cleanly

---

## 6. Issues
Use GitHub Issues for:
- Bug reports
- Documentation improvements
- Feature requests
- Clarification questions

Please be specific and include examples when possible.

---

## 7. Code of Conduct
Be respectful, constructive, and collaborative.  
REID is a long‑arc project intended to support clarity and shared
understanding across domains and contributors.

---

Thank you for helping improve REID.
