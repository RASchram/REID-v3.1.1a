Reasoning Efficiency Examples
How REID Reduces Steps, Drift, and Compute Load
This document demonstrates how REID improves reasoning efficiency
through compression, consolidation, and invariant‑based structure.
Each example shows a “traditional reasoning path” compared to a
“REID‑structured path.”

The goal is to illustrate how REID produces clearer outcomes with
fewer steps and lower cognitive or computational cost.

1. Example: Planning a Multi‑Step Task
Traditional Reasoning Path
Identify the task

Clarify requirements

Clarify constraints

Re‑clarify missing details

Break into steps

Reorder steps

Identify dependencies

Re‑evaluate constraints

Finalize plan

This often results in long chains of thought, repeated clarifications,
and unnecessary loops.

REID‑Structured Path
Identify state (entities, roles, resources)

Identify forces (constraints, pressures, incentives)

Identify horizon (goal, timeline, acceptable outcomes)

Identify flows (movement of time, energy, resources)

Produce plan directly from invariants

REID removes the need for repeated clarifications by using a stable
structure that applies to any task.

2. Example: Reducing Cognitive Drift
Traditional Path
Reasoning drifts when:

Context changes

Emotional state shifts

New information appears

Time passes

Multiple domains overlap

This leads to inconsistent conclusions.

REID Path
REID maintains stability through:

Substrate coherence

Horizon anchoring

Invariant structure

Reduced expansion

Stable long‑arc logic

This prevents drift even when the domain or context changes.

3. Example: Faster Decision Making
Traditional Path
Gather information

Compare options

REID Path

This reduces the number of steps required to reach a stable decision.
